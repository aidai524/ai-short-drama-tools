# Storyboard Splitter / 短剧分镜拆解器

将详细的中文短剧剧本，拆解为三种不同风格的分镜剧本，覆盖从阅读讨论到 AI 生成到专业拍摄的全链路需求。

---

## 三种风格总览

| 风格 | 代号 | 核心特征 | 适用场景 |
|------|------|----------|----------|
| **叙事版** | Narrative | Markdown叙述体，可读性强，每个分镜独立小节 | 编剧/导演阅读讨论、剧本评审 |
| **生成版** | Generative | 秒级时间段 + @素材引用 + 声音设计 + 尾帧描述，Seedance/Kling API 就绪 | AI 视频生成（Seedance 2.0 / Kling v3） |
| **制片版** | Production | 精确镜头语言表格，含景别/运镜/同期声/闪回标注 | 专业拍摄/动画制作/剧组执行 |

---

## 工作流程

### Phase 1: 接收并分析剧本

1. **接收剧本** — 用户提供的详细剧本文本（支持 .txt / .md / .docx 或直接粘贴）
2. **识别结构** — 自动识别：
   - 集数（第X集）
   - 场次（X—X，日/夜，内/外，地点）
   - 人物列表及角色关系
   - 台词、动作描写（△开头）、内心独白（os）、字幕标注
   - 场景转换标记
3. **生成全局信息** — 提取并汇总：
   - 剧名
   - 画面风格声明（色调、光影氛围）
   - 场景清单（去重后的所有场景）
   - 角色设定表（姓名、身份、外貌、性格关键词）
   - 画面比例建议（短剧默认 9:16）

### Phase 2: 拆解分镜

4. **逐场拆分** — 对每个场次，按以下规则拆分为独立分镜：
   - **动作节点** — 每个 △ 标记的动作描写 → 独立分镜或与相邻台词合并
   - **台词轮次** — 同一说话人的连续台词合并，不同说话人切换 → 新分镜
   - **内心独白** — OS/内心独白标记 → 独立分镜（标注为无声镜头）
   - **场景转换** — 地点/时间变化 → 强制断点
   - **情绪转折** — 角色情绪显著变化 → 新分镜
   - **闪回/前世线** — 闪回标记 → 独立分镜组（标注色调变化）
5. **为每个分镜分配**：
   - 唯一镜号（格式：`场次-序号`，如 `1-1-03`）
   - 时长估算（参考下方时长指南）
   - 景别（参考 @references/cinema-glossary.md）
   - 运镜方式（参考 @references/cinema-glossary.md）
   - 人物列表
   - 情绪基调

### Phase 3: 按风格输出

根据用户选择的风格（默认输出全部三种），按对应模板格式化。

---

## 风格一：叙事版（Narrative）

**定位**：可读性最强的阅读本，适合编剧、导演在电脑/平板上阅读讨论。

### 输出格式

```markdown
# [剧名] — 第X集分镜剧本

## 画面风格
[整体画面风格声明：色调、光影、氛围]

## 场景信息
- **场景A**：[地点1]（[环境描述]）
- **场景B**：[地点2]（[环境描述]）

## 角色设定
- **[角色名]**（[身份]）：[外貌描述]，[性格关键词]

---

## 分镜 [镜号] | [简要标题]

**场景**：[场景名称]
**镜号**：[场次-序号]
**时长**：[X]s
**景别**：[景别中文]（[英文缩写]）
**运镜**：[运镜方式]
**人物**：[出场人物]

**画面描述**：
[2-4句详细的画面描述，包含角色动作、表情、环境细节、光影效果]

**台词**：
- [角色名]（[情绪/动作标注]）："[台词内容]"
- [角色名]（[情绪/动作标注]）："[台词内容]"

**内心独白（OS，不张嘴）**：
"[内心独白内容]"

**情绪**：[主要情绪关键词]

---

## 分镜 [镜号] | [简要标题]
...
```

### 叙事版规则

1. 每个分镜是一个独立的 `---` 分隔的 Markdown 小节
2. **画面描述**用完整的叙述性中文，描述"看到了什么"
3. **台词**保留原文，括号内标注说话时的情绪和动作
4. **内心独白**单独列出，标注"不张嘴"
5. **情绪**用 2-3 个关键词概括
6. 文件末尾附 **分镜汇总表**（表格形式列出所有分镜的镜号/时长/场景/人物/核心事件）
7. 文件末尾附 **备注**（制作建议、色调区分、结尾悬念设计等）

---

## 风格二：生成版（Generative）

**定位**：直接喂给 Seedance 2.0 / Kling v3 API 的 AI 视频生成格式。带时间戳、素材引用、声音设计。

### 输出格式

```markdown
# [剧名] — 第X集 · 生成版分镜

## 画面风格声明
[风格描述]，[色调]，[分辨率]，[秒数]，[画面比例]，[氛围关键词]

## 素材映射
- @图片1 — [描述] → [文件路径或URL]
- @图片2 — [描述] → [文件路径或URL]

---

## 分镜 [镜号]：[简要标题]

[时间段起始]—[时间段结束]秒：[阶段标题] — [景别]，[场景描述]（@[素材引用]），[环境细节]，[运镜方式]，[氛围]

[时间段起始]—[时间段结束]秒：[阶段标题] — [角色描述]，[动作描述]，[运镜切换]

[时间段起始]—[时间段结束]秒：[阶段标题] — [情感动作]，[表情描述]，[运镜]

...

【声音】[BGM描述] + [环境音效列表] + 对白："[关键台词]"
【参考】@[素材引用列表及说明]

尾帧描述
主体：[尾帧主体描述]
背景：[尾帧背景描述]
光线：[光线描述]
构图：[构图描述]
氛围：[氛围关键词]

---

## 分镜 [镜号]：[简要标题]
...
```

### 生成版规则

1. **文件开头声明**整体画面风格（动画/真人风格、色调、分辨率、画面比例）
2. **每个分镜按时间段拆分**（如 `0-3秒`、`3-6秒`），每段 3-6 秒
3. **@素材引用** — 将角色参考图、场景参考图、道具参考图用 `@图片N` 标注，文件开头有素材映射表
4. **【声音】段落** — 每个分镜末尾列出：
   - BGM 描述（风格+乐器+情绪）
   - 环境音效列表（具体声音）
   - 关键对白（带角色名和情绪标注）
5. **尾帧描述** — 每个分镜末尾提供结构化的尾帧描述（主体/背景/光线/构图/氛围）
6. **内心独白**处理为"silent internal monologue"，描述表情变化而非嘴部动作
7. **闪回/前世线**标注色调变化（如"色调偏暗泛黄，轻微噪点"）
8. **连续性约束** — 每个时间段末尾追加 `same outfit, same hairstyle, same location layout, same lighting`
9. 文件末尾附 **素材需求汇总表**（列出所有需要的参考图/场景图/道具图）

### 生成版 Prompt 工程规则（关键 — 决定 AI 视频生成质量）

以下规则在将生成版分镜转化为 Seedance/Kling API prompt 时**必须强制执行**：

#### R1: 角色首段声明（每段 prompt 开头必须）

每个视频片段的 prompt **开头**必须包含该段出场角色的声明块，格式：

```
[角色英文名] (asset://[虚拟人物ID], [性别] [年龄] [外貌关键词] [服装描述])
```

**示例**：
```
Cheng Shizhong (asset://asset-20260225015021-bkztc, male 50s overweight man in dark suit)
Su Jiajia (asset://asset-20260225014926-vdnsx, female 20s beautiful young woman in casual top)
Jiang Zhifeng (asset://asset-20260225014926-c6947, female 20s graceful woman long black hair in light-colored dress)
```

**原因**：Seedance 模型按出场顺序和戏份权重分配角色外貌。明确声明能防止角色混淆、性别错误、服装不一致。

#### R2: 角色每次出场标注 asset://

prompt 中每次提到角色名字时，**必须附带 asset:// ID**：

```
✅ Jiang Zhifeng (asset://asset-20260225014926-c6947) says aloud in Mandarin: "..."
✅ Su Jiajia (asset://asset-20260225014926-vdnsx) grabs her arm...

❌ Jiang Zhifeng says aloud...  （缺少 asset 标注，模型可能混淆角色）
```

**原因**：Seedance 模型通过 prompt 中的 asset:// ID 与 content 中的 reference_image 做交叉匹配。缺少标注时模型按戏份权重自行分配，多角色场景极易混淆。

#### R3: 正确使用性别代词

prompt 中描述角色动作时，**必须使用与角色性别一致的英文代词**：

| 角色性别 | 代词 | 示例 |
|----------|------|------|
| 女性 | she / her | "She crosses her arms", "her foot slips" |
| 男性 | he / his / him | "He waves his hands", "pushes him" |

**严格禁止**：女性角色使用 he/him/his，或男性角色使用 she/her。

**原因**：Seedance 模型会根据代词推断角色声音性别。女性角色用 he → 生成男声；男性角色用 she → 生成女声。

#### R4: 禁止字幕/画面文字

每个 prompt **末尾必须追加**：

```
IMPORTANT: DO NOT display any subtitles, captions, or on-screen text overlay. No text should appear on screen at any time.
```

**原因**：Seedance 默认在检测到对白时自动生成字幕。除非剧本明确要求显示字幕，否则一律禁止。

#### R5: 声音性别声明

每个 prompt **末尾必须追加**角色声音性别声明：

```
[角色英文名](asset://[ID])是[女性/男性]，声音必须是[描述]。
```

**示例**：
```
Jiang Zhifeng(asset://asset-20260225014926-c6947)是女性，声音必须是年轻女性柔和嗓音。
Cheng Shizhong(asset://asset-20260225015021-bkztc)是中年男性，声音低沉沙哑。
Su Jiajia(asset://asset-20260225014926-vdnsx)是年轻女性，声音清脆甜美。
```

**原因**：即使使用正确代词，模型仍可能在复杂场景中混淆声音。显式声明是双重保险。

#### R6: 帧连续性保障（防跳帧）

Seedance 单段视频最长支持 15 秒。时长不是问题，**动作之间的过渡描述**才是关键。以下是防止帧不连续的规则：

##### R6.1 单一空间连续（最重要）

**每个 prompt 只描述一个连续的空间动作流**，禁止空间跳跃：

```
✅ Jiang Zhifeng walks from the bed to the door, camera tracking behind her.
   Su Jiajia leaps off the bed, sprints after her, grabs her arm from behind.
   （动作自然衔接：江走→苏追→苏抓，空间连续）

❌ Jiang Zhifeng walks toward the door. Then she lies on the floor bleeding.
   （从门口突然跳到倒地，中间没有任何过渡，导致5-6秒处跳帧）
```

##### R6.2 用 "begin → transition → end" 结构描述动作

每个独立动作用三段式描述，把"怎么过去"写出来：

```
✅ The scene begins with Jiang Zhifeng standing by the bed pointing accusingly.
   Transitioning to her turning sharply and walking briskly toward the door.
   Ending with her hand reaching for the door handle.

❌ Jiang Zhifeng points at the bed. She walks to the door.
   （两个动作之间没有过渡描述，模型会硬切）
```

##### R6.3 保持单一摄影机视角

**一个 prompt 内不要切换机位**（如从中景切到鸟瞰再切回中景）。镜头切换=跳帧。

```
✅ Camera stays at medium wide shot, slowly pushing in as the argument escalates.
   （单一机位，推近，画面流畅）

❌ Wide shot of the room. Cut to bird's eye view. Back to close-up.
   （三次切机位，模型无法平滑过渡，必然跳帧）
```

##### R6.4 位置锚定

描述每个动作时，标注角色在空间中的**相对位置**：

```
✅ Su Jiajia (standing by the bed, left side of frame) lunges toward
   Jiang Zhifeng (near the door, right side of frame).
   （明确双方位置，模型能正确计算运动轨迹）

❌ Su Jiajia grabs Jiang Zhifeng.
   （没有位置信息，模型可能让两人瞬移）
```

##### R6.5 何时拆分 vs 不拆分

| 场景 | 策略 | 原因 |
|------|------|------|
| 同一空间连续动作流 | **不拆**，15 秒一段 | 动作自然衔接，无需拆 |
| 动作之间有位置跳跃（从门口到床底） | **拆成两段**，尾帧→首帧链接 | 位置跳太远，模型无法平滑过渡 |
| 需要切换机位（全景→特写） | **拆成两段** | 切机位=强制跳帧 |
| 复杂互动（打斗、追逐、多人撕扯） | **拆成两段** | 动作太复杂，模型丢失细节 |

#### R7: 内心独白特殊处理

```
✅ Jiang Zhifeng silent internal monologue, lips firmly closed, contemplative expression, eyes unfocused in thought: [中文独白内容]
❌ Jiang Zhifeng thinks: "..."  （模型可能让角色张嘴说话）
```

独白中的中文内容**不翻译为英文**，但前面必须标注 `silent internal monologue, lips firmly closed`。

## 风格三：制片版（Production）

**定位**：精确到单个镜头的专业拍摄分镜表。含景别代码、运镜术语、同期声标注。可直接用于剧组拍摄执行。

### 输出格式

```markdown
# [剧名] — 第X集 · 制片版分镜表

## 色调方案
[按场次/情绪阶段声明色调变化]
例：前期暖黄（怀旧梦幻）→ 中期冷蓝（残酷现实）→ 后期苍白（真相揭晓）

## 场次索引
| 场次 | 地点 | 时间 | 内/外 | 主要人物 | 镜号范围 |
|------|------|------|-------|----------|----------|
| 1-1 | 酒店套房 | 日 | 内 | 程世忠、苏佳佳、江止风 | 1-01 ~ 1-07 |

---

## 第一场：[场标题]（[总时长]）

| 镜号 | 景别 | 运镜 | 画面描述 | 音效/对白 | 时长 |
|------|------|------|----------|-----------|------|
| 1 | [景别] | [运镜] | [画面描述] | [音效/对白] | [X]s |
| 2 | [景别] | [运镜] | [画面描述] | [音效/对白] | [X]s |
| ... | ... | ... | ... | ... | ... |

本场小结： [一场戏的叙事目标和情绪总结]

---

## 第二场：[场标题]（[总时长]）
...
```

### 制片版规则

1. **每个场次一张表**，场次之间用 `---` 分隔
2. **表头固定**：镜号 | 景别 | 运镜 | 画面描述 | 音效/对白 | 时长
3. **景别使用标准术语**（参考 @references/cinema-glossary.md）：
   - 大特 / ECU（极端特写）
   - 特写 / CU
   - 近景 / MCU
   - 中景 / MS
   - 中远景 / MLS
   - 远景 / WS
   - 大远景 / EWS
   - 过肩 / OTS
   - 主观 / POV
4. **运镜使用标准术语**：
   - 固定 / static
   - 推 / push-in
   - 拉 / pull-out
   - 摇 / pan
   - 俯仰 / tilt
   - 跟拍 / tracking
   - 环绕 / orbit
   - 手持 / handheld
   - 升降 / crane
   - 快切 / whip pan
   - 渐亮 / fade in
   - 渐暗 / fade out
   - 闪回 / flashback
   - 抓拍 / grab shot
   - 跟焦 / follow focus
   - 急推 / fast push-in
5. **画面描述**简洁精确，1-2 句话，描述核心视觉内容
6. **音效/对白列**标注：
   - 环境音：`[音效描述]`
   - 同期声：`[角色名]："[台词]"`（标注"同期声"）
   - BGM：`[BGM描述]`（仅在变化时标注）
   - 无声：留空或标注"静默"
7. **闪回**在运镜列标注"闪回"，画面描述中注明色调变化
8. 每场末尾附 **本场小结**（1-2 句话总结该场叙事目标和情绪弧线）
9. 文件末尾附 **全剧分镜统计**（总场次数、总分镜数、总时长、各场景用时分布）

## 时长估算指南

| 动作类型 | 建议时长 | 说明 |
|----------|----------|------|
| 标准对话场景（2-3句台词） | 8-10s | 含动作反应 |
| 单一连续动作流 | 10-15s | 如：走廊走→推门→进入房间，动作自然衔接 |
| 复杂互动（撕扯/打斗） | 拆成 2 段 × 8s | 位置跳跃大，拆开后用尾帧链接 |
| 环境建立 + 角色登场 | 10-12s | 先建立场景，再角色入画 |
| 闪回插入 | 6-8s | 短促有力 |
| 情绪特写 / 反应镜头 | 4-6s | 聚焦表情变化 |

> **核心原则**：时长不是限制（最长 15 秒），关键是每个 prompt 内的动作必须在**同一空间连续**，描述清楚过渡过程。

## 剧本解析规则

### 输入识别标记

| 标记 | 含义 | 解析动作 |
|------|------|----------|
| `第X集` | 集数分界 | 按**集**拆分输出文件 |
| `X—X，日/夜，内/外，[地点]` | 场次头 | 识别场景，开始新场次 |
| `△[动作描写]` | 动作/画面指示 | 转为画面描述 |
| `角色名（[情绪]）：[台词]` | 角色台词 | 保留原文，标注情绪 |
| `os：[内容]` 或 `（OS，不张嘴）` | 内心独白 | 独立分镜，标注无声 |
| `【字幕——角色名/身份】` | 角色首次登场字幕 | 分镜中标注字幕叠加 |
| `【闪回】`/`【闪出】` | 闪回区间 | 标注色调变化，独立分镜组 |
| `【镜头黑屏，画面切换】` | 转场 | 强制分镜断点 |
| `△话落`/`△说着` | 动作衔接 | 合并到台词分镜 |
| `切镜头` | 镜头切换 | 新分镜 |

### 剧本输入格式支持

1. **标准格式**（推荐）— 含场次头、△标记、台词格式的结构化剧本
2. **口语化格式** — 自由叙述的剧本，需更多推断
3. **Word文档** (.docx) — 自动提取文本后解析

## 常见问题

### Q: 一段剧本应该拆成多少个分镜？
平均每页剧本（约 500 字）拆 8-15 个分镜。节奏快的动作戏可拆更多（15-20），对话戏可拆较少（8-12）。

### Q: 三种风格有什么区别？
- **叙事版**：给**人**读的，像小说一样有画面感，适合讨论和评审
- **生成版**：给**AI**读的，带时间戳和素材引用，直接生成视频
- **制片版**：给**剧组**用的，精确镜头语言，直接指导拍摄

### Q: 可以只输出一种风格吗？
可以。指定风格名即可。不指定则默认输出全部三种。

### Q: 素材引用（@图片）从哪来？
生成版中的 @图片 引用是占位符。用户需要后续提供参考图，或使用 ai-image-generation skill 生成场景参考图，再填入素材映射表。

### Q: 支持什么画面比例？
短剧默认 9:16（竖屏）。也可指定 16:9（横屏）、21:9（电影宽屏）等。在画面风格声明中标注。
